# Review Backend 后端项目审查

全面审查后端服务项目的代码质量和架构设计。

## 审查清单

### 1. 架构设计

- [ ] 分层清晰 (Controller/Service/Repository)
- [ ] 依赖注入正确使用
- [ ] 接口抽象合理
- [ ] 模块边界清晰

### 2. API 设计

- [ ] RESTful 规范或 GraphQL 规范
- [ ] 统一的响应格式
- [ ] 合理的 HTTP 状态码
- [ ] API 版本控制
- [ ] 接口文档完整 (OpenAPI/Swagger)

### 3. 数据库

- [ ] 表结构设计合理
- [ ] 索引使用正确
- [ ] 避免 N+1 查询
- [ ] 事务边界清晰
- [ ] 数据库迁移脚本完整

### 4. 安全性

- [ ] 输入验证和清理
- [ ] SQL 注入防护 (参数化查询)
- [ ] 认证授权完善
- [ ] 敏感数据加密
- [ ] 日志不泄露敏感信息

### 5. 错误处理

- [ ] 统一的错误处理机制
- [ ] 错误信息对用户友好
- [ ] 详细的内部日志
- [ ] 优雅的降级处理

### 6. 可观测性

- [ ] 结构化日志
- [ ] 请求链路追踪
- [ ] 关键指标监控
- [ ] 健康检查端点

## 审查报告格式

