# Review Desktop 桌面端项目审查

全面审查桌面客户端应用的代码质量。

## 审查清单

### 1. 架构设计

- [ ] 主进程与渲染进程分离 (Electron)
- [ ] 前后端通信机制清晰
- [ ] 模块化设计合理
- [ ] 插件系统设计 (如有)

### 2. 安全性

- [ ] 禁用 nodeIntegration (Electron)
- [ ] 启用 contextIsolation
- [ ] IPC 通信验证输入
- [ ] 避免加载远程不可信内容

### 3. 性能优化

- [ ] 启动时间优化
- [ ] 内存占用合理
- [ ] 避免主线程阻塞
- [ ] 大文件异步处理

### 4. 跨平台兼容

- [ ] Windows/macOS/Linux 测试
- [ ] 文件路径处理正确
- [ ] 系统 API 兼容性
- [ ] 原生菜单和快捷键

### 5. 用户体验

- [ ] 支持系统主题
- [ ] 窗口状态记忆
- [ ] 托盘图标功能
- [ ] 自动更新机制

### 6. 打包发布

- [ ] 代码签名配置
- [ ] 安装包大小优化
- [ ] 自动更新配置
- [ ] 崩溃上报集成
